### Task 2.1: Implement `src/services/sesion.service.ts`
- **Action**: Created an in-memory session singleton (`SesionService`) that registers active operator/line sessions.
- **Details**:
  - Enforces the strict rule that an operator can only have one active session at a time. If they open a session on a new line, their previous active session is automatically terminated.
  - Maps `lineaProduccionId` to an active session context (`ActiveSession`) containing `usuarioId`, `lineaProduccionId`, `articuloId`, `pasadaId`, and `connectedAt`.

### Task 2.2: Implement `src/services/pasada.service.ts`
- **Action**: Created the `PasadaService` extending the generic `BaseService`.
- **Details**:
  - Implements `iniciarPasada()` with a pessimistic database write-lock (`SELECT ... FOR UPDATE` via `LockMode.PESSIMISTIC_WRITE` on `LineaProduccion`) to serialize pasada creation on the line.
  - Guarantees sequential unique numbering (`numero`) starting from 1 for each line-article combination without concurrency duplication conflicts.
  - Sets the associated `pasadaId` in the in-memory active session when started, and clears it when completed.

### Task 2.3: Implement `src/services/muestra.service.ts`
- **Action**: Created the `MuestraService` extending `BaseService`.
- **Details**:
  - Performs real-time validation of weight packets (`pesoNeto`) against limits defined in `RutaPasadaEtapa` (`pesoMinimo <= pesoNeto <= pesoMaximo` -> `ok`, else `fuera_de_rango`).
  - Enforces sequential stage progression in ascending order of `orden`: rejects sample registrations for any stage if preceding stages have not met their required OK sample count.
  - Integrates out-of-range (`fuera_de_rango`) samples: persists them for full traceability, but does not count them toward stage completion.
  - Completes the entire weight run: transitions the `Pasada.estado` to `completa` and sets the `horaCierre` timestamp when the final sample completes the final stage of the route.

### Task 2.4: Restrict soft-delete and updates on completed records
- **Action**: Overrode the default `update()` and `softDelete()` methods in both `PasadaService` and `MuestraService`.
- **Details**:
  - Blocks and rejects any modification or soft-deletion of `Pasada` records that are completed (`estado = completa`).
  - Blocks and rejects any modification or soft-deletion of `Muestra` records belonging to a completed `Pasada`.

### Task 2.5: Write focused Vitest tests
- **Action**: Created separate test files for both session registry and sample validation/stage logic.
- **Details**:
  - `src/services/sesion.service.test.ts`: Pure unit tests validating in-memory operator session uniqueness constraints, login/logout transitions, and properties.
  - `src/services/muestra.service.test.ts`: Robust database integration tests verifying lock-based sequential pasada numbers, weight limits classifications, sequential stage requirements, final complete transitions, and soft-delete/update blocks on finished records.
